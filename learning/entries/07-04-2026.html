It looks like there are 3 terms for each element:

  • Margin: The space outside the border of an element.
  • Padding: The space inside the border of an element.
  • Border: The line that surrounds an element.

The image here is a little misleading, there are only 2 space elements, margin and padding. Border is the line thickness which also in a way creates some space. But shown as 3 elements in the image.

Separately, the p element is filling the vertical space with the content, but if it is forcefully increased, the content is aligned to the top (like adding it to a div, which has bigger elements in the page like a h2)

It looks like this is caused by flexbox behavior. Flexbox stretches stuff by default

Flexbox tries to align items in the cross axis by default, this stretches items to align vertically both top and bottom.

To fix this, we can use align-items: flex-start; to align items to the top of the container.

Used the same method to fix the streteching of side by side image above as well.
